# 07_UserStories_UseCases

Short description
- Collection of user stories and use cases mapping user needs to features and acceptance criteria.

Objective
- Provide clear, testable user stories and use cases to populate a feature backlog.

Contents
- User story template: As a [role], I want [goal], so that [benefit]. Include acceptance criteria.

Steps
1. Gather requirements via interviews and workshops.
2. Draft user stories and prioritize them.
3. Translate stories into use cases and flows.
4. Add acceptance criteria and example test cases.

Deliverables
- user_stories.md with prioritized stories.
- Use case diagrams or flow maps if applicable.
- README explaining how to use and maintain the backlog.
